# 作業報告（2026-01-07〜2026-01-11）

## 概要

対照学習モデル（MotionCLIP）のハイパーパラメータ最適化と、強化学習でのスタイル報酬実験を行った。

---

## 1. 解決した問題：座標系の統一

### 問題
- ロボット（H1）の動きをモーションデータセット（HoYo）と対応付ける際、**world座標の回転（yaw）が混入**すると特徴量が潰れてしまい、正しくスタイルを学習できなかった。

### 解決策
- **yaw不変**な座標系に統一。ロボットの前面方向（Y-Z平面）を2D投影してMotionCLIPに入力するようにした。
- 前処理も **2秒間・50Hz** で統一し、関節の対応付けを固定オフセットで管理。

## 2. 対照学習（MotionCLIP）の再実験

### 目的
ロボットの動き（モーション）とテキスト（「ゆっくり歩く」「元気に走る」など）を同じ埋め込み空間に配置し、テキストでスタイルを指定できるようにする。

### 実施内容
- **Optuna**を使い、20回の試行でハイパーパラメータを自動探索。
- 学習は2段階：
  1. **Freeze段階**：MotionCLIPの事前学習済み重みを固定し、新規レイヤーのみ学習
  2. **Full段階**：全体をfine-tuning

### 評価指標

| 指標 | 説明 |
|------|------|
| **Motion→Text R@k** | モーションを入力し、正しいテキストが上位k件に含まれる割合 |
| **Text→Motion R@k** | テキストを入力し、正しいモーションが上位k件に含まれる割合 |
| **Silhouette Score** | 同じスタイルのデータが埋め込み空間で近くにまとまっているか（-1〜1、高いほど良い） |

### 評価結果（学習終了時 step=14,000）

| 指標 | 値 |
|------|-----|
| **Text→Motion R@1** | 72.7% |
| **Text→Motion R@3** | 90.9% |
| **Text→Motion R@5** | 100% |
| **Motion→Text R@1** | 43.5% |
| **Motion→Text R@3** | 77.4% |
| **Motion→Text R@5** | 91.9% |
| **Silhouette Score** | -0.031 |

- Text→Motion（テキストからモーションを検索）は高精度で、R@5で100%を達成。
- Motion→Text（モーションからテキストを検索）はR@1が43.5%とやや低い。モーションの多様性に対しテキストラベルが粗いことが原因と考えられる。
- Silhouetteがマイナスなのは、クラス間の重なりがあることを示す。

### 最適パラメータ（Optunaで探索）

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 学習率 | 1.7e-5 | 小さめの値で安定学習 |
| Weight Decay | 4.2e-6 | 過学習防止の正則化 |
| Temperature | 0.023 | 対照学習の温度パラメータ（小さいと厳しく判定） |
| 学習ステップ | 14,000 | Full段階のステップ数 |

## 5. 分析

### 5.1 速度とオノマトペの意味的関係

| 期待 | オノマトペ | 実際の速度 | 評価 |
|------|-----------|-----------|------|
| 遅い | とぼとぼ | 0.583 (速い方) | ❌ 逆転 |
| 遅い | のろのろ | 0.549 (中間) | ❌ 逆転 |
| 速い | せかせか | 0.542 (中間) | ❌ 逆転 |
| 普通 | どっしどっし | 0.422 (最遅) | ❌ 逆転 |

**原因:** `StyleModule._prepare_centered_2d` でグローバル位置情報（移動量）を削除しているため、MotionCLIP encoder が速度情報を受け取れていない。

### 5.2 TextSim / CentrSim の分析

| ランク | オノマトペ | TextSim | CentrSim | 評価 |
|--------|-----------|---------|----------|------|
| 1位 | よろよろ | 0.348 | 0.428 | ✅ 良好 |
| 2位 | ぶらぶら | 0.322 | 0.387 | ✅ 良好 |
| 3位 | てくてく | 0.313 | 0.316 | △ |
| 最下位 | せかせか | 0.120 | 0.340 | ❌ TextSim極端に低い |

**考察:** よろよろ、ぶらぶらは比較的高いスタイル類似度。せかせかはTextSimが極端に低く、学習がうまくいっていない可能性。

### 5.3 姿勢崩れの問題（重大）

**観測された問題:** 動画確認の結果、**右肩が左肩に対して顕著に低い**非対称な姿勢で歩行。

| 指標 | 全スタイル平均 | 正常値 |
|------|--------------|--------|
| Roll | 0.38〜0.43 rad (22〜25°) | ≈ 0 rad |

**原因分析:**
1. **Base policy は正常** — style なしでは姿勢崩れなし（確認済み）
2. **Style tracking が強すぎる（weight=5.0）** — 姿勢ペナルティ（-0.2）に対して強すぎる
3. **最終層のみ unfreeze** — 調整の自由度が低く、style tracking に引っ張られた

### 5.4 HOYO Error の実装問題

#### 改善点（Yaw補正追加）

| 修正前 | 修正後 |
|--------|--------|
| 0.93〜1.55 | **0.58〜0.68** |

Yaw補正を追加したことで、ロボットの向きに依存しない一貫した比較が可能になった。

#### 残存する問題

**分散が小さすぎる:**
- 全オノマトペで 0.58〜0.68 の範囲（差が 0.1 しかない）
- 「どのオノマトペでも同じくらいの動き」になっている可能性

**実装上の問題:**
- 5サンプルの中から最も近いものとの距離を返す
- どんな動きでも「それなりに近い」参照が見つかってしまう
- **識別性（そのオノマトペらしさ）を測れていない**

**改善案:**
1. 他クラスとの距離も計算して識別性を評価
2. 最小距離ではなく平均距離を使用
3. サンプル数を増やす（5 → 20）

## 6. 評価指標の説明と信頼性

### 6.1 各指標の説明

| 指標 | 説明 | 理想値 |
|------|------|--------|
| **Vel X** | ロボットの前進速度（Body Frame、進行方向基準） | コマンド速度と一致 |
| **Roll** | 左右の傾き（横転角）。クォータニオンからオイラー角変換 | 0 rad |
| **Pitch** | 前後の傾き（縦揺れ角） | 0 rad |
| **TextSim** | テキスト類似度。ロボット動きとオノマトペ埋め込みのcos類似度 | 1.0 |
| **CentrSim** | セントロイド類似度。ロボット動きとHOYOクラス重心の類似度 | 1.0 |
| **HOYO Error** | HOYOデータとの形状誤差。DTW距離（5サンプル中の最小値） | 0.0 |
| **転倒率** | 転倒回数 / エピソード数（転倒でリセット） | 0% |

### 6.2 指標の詳細

#### TextSim（テキスト類似度）

```
z_agent = MotionCLIP(ロボットの動き)    # (512,)
z_text  = TextEncoder(オノマトペ) → Projection  # (512,)
TextSim = cosine_similarity(z_agent, z_text)
```

- **意味**: ロボットの動きがオノマトペの「意味」にどれだけ近いか
- **範囲**: -1.0 〜 1.0（高いほど良い）
- **備考**: 対照学習で学習した共有埋め込み空間での類似度

#### CentrSim（セントロイド類似度）

```
z_agent    = MotionCLIP(ロボットの動き)  # (512,)
z_centroid = mean(MotionCLIP(HOYOサンプル))  # そのオノマトペの全サンプル平均
CentrSim   = cosine_similarity(z_agent, z_centroid)
```

- **意味**: ロボットの動きがHOYOデータの「典型的な動き」にどれだけ近いか
- **範囲**: -1.0 〜 1.0（高いほど良い）
- **備考**: TextSimより具体的なモーション形状との類似度

#### HOYO Error（形状誤差）

```
robot_motion = get_buffer_for_hoyo_comparison()  # (T, 14, 2) 2D関節座標
hoyo_samples = [sample1, ..., sample5]  # 各オノマトペから5サンプル
HOYO_Error   = min([DTW(robot_motion, sample) for sample in hoyo_samples])
```

- **意味**: ロボットの関節軌跡がHOYOデータにどれだけ近いか（時系列形状の比較）
- **範囲**: 0.0 〜 ∞（低いほど良い）
- **問題点**: 最小値を取るため識別性が低い（どんな動きでも「近い」参照が見つかる）

### 6.3 信頼性まとめ

| 指標 | 信頼性 | 理由 |
|------|--------|------|
| Vel X | ✅ 高 | Body Frame で正しく計測 |
| TextSim | △ 中 | 意味的類似度を測れるが、姿勢崩れの影響あり |
| CentrSim | △ 中 | 同上 |
| HOYO Error | ❌ 低 | 識別性がない、全オノマトペで似た値 |
| Roll/Pitch | ✅ 高 | 姿勢崩れの定量化に有効 |
| 転倒率 | ✅ 高 | 安定性の直接的指標 |

### 評価コードの速度計測について

```python
# Body Frame 速度を使用（回転を考慮）
base_lin_vel = getattr(robot.data, "root_lin_vel_b", robot.data.root_lin_vel_w)
```

- `root_lin_vel_b`: ロボットのローカル座標系での速度（X = 前方向）
- ロボットが回転しても、常に「進行方向」への速度を正しく計測
