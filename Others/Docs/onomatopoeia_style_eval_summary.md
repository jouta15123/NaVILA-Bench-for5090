# 報告資料案：オノマトペ・スタイル付与RL 評価設計まとめ

## 1. 評価の目的
本研究では、オノマトペ指示によるスタイル付与を「意味整合・見た目一致・タスク性能」の3点から評価する。
特に、**「表現学習（対照学習）」**と**「行動獲得（強化学習）」**の評価を切り分け、どこに課題があるかを明確に分離して報告することを目指す。

---

## 2. 対照学習（CL）の評価：表現の健全性
オノマトペとモーションの埋め込みが、潜在空間で正しく対応しているかを評価する。

### 主要指標（Retrieval）
> **直感的なイメージ**: Google検索で「ヨタヨタ歩く」と入れたとき、正解の動画が何番目に出てくるか？

- **$R@K$ (Recall at K)**
  - 「トップK件（1位, 3位, 5位）以内に正解が含まれている確率」。
  - **高いほど良い**（検索精度が高い）。
- **MedR (Median Rank)**
  - 「正解が出てくる順位の真ん中（中央値）」。
  - **低いほど良い**（例: MedR=1 なら、だいたい1位に正解が出てくる優秀なモデル）。

### 補助指標（Clustering）
> **直感的なイメージ**: 散布図上で、同じスタイルの点がちゃんと固まっているか？

- **Silhouette係数**
  - -1〜+1 の値をとる。
  - **+1に近い**: スタイルごとに綺麗に分かれている（「チョコチョコ」と「ドタドタ」が混ざっていない）。
  - **0に近い**: 境界が曖昧で混ざっている。

### 現状ステータス
- [x] Prototypeベースの分類精度（Acc@1/3）の実装。
- [ ] サンプル間の全検索（$R@K$ / MedR）の実装。
> ※全件Retrievalは論文（MotionCLIP, T2M等）で標準的に使われる最も厳密な指標です。

---

## 3. 強化学習（RL）の評価：行動としてのスタイル
獲得したポリシーが、指示通りかつタスクを完遂できているかを評価する。

### 評価指標一覧

| カテゴリ | 指標名 | 評価内容 |
| :--- | :--- | :--- |
| **Style Fidelity** | 外部分類器精度 (Top-1/F1) | 別の学習済み分類器によるスタイル判定。<br>※リーク防止のためRL学習用とは分離。 |
| **Teacher Consistency** | $\cos(z_{motion}, z_{teacher})$ | 生成された挙動が教師モーションの埋め込みとどれだけ近いか。 |
| **Task Performance** | 成功率 / 到達時間 | 移動タスク等の本来の目的が達成できているか。 |
| **Stability** | 転倒率 | 物理的な安定性。 |

### 評価の工夫
- **自己採点の回避**: 報酬計算用とは別の「評価専用エンコーダ」を用意する。
- **計算詳細**: $z_{motion}$ は 60f 窓・stride=10f で計算し、平均と分散を併記する。

> **Note: なぜ分類器が必要か？（模倣誤差との違い）**
> *   **模倣誤差 (4章)**: 教師データを「正確にトレースできているか」を見る。
> *   **分類器 (3章)**: 物理制約による姿勢変化や、歩行速度が変わった場合でも「スタイル（雰囲気）が保たれているか」を見る。
> RLではバランス維持のために動きを変形させる必要があるため、両方の指標を併用する。

---

## 4. 見た目の模倣一致（補助指標）
HOYOデータセットの教師動作に対して、物理的な追従度を確認する。

- **HOYO 2D Imitation ($r_I$)**
  - 関節位置・速度・重心（COM）の誤差。
- **位相ずれ対策**
  - DTW（Dynamic Time Warping）を用いた時間軸補正。
- **幾何学的整合性**
  - 座標系依存を避けるため、骨方向（単位ベクトル）を用いた誤差評価を併用。

## 5. 実験プロトコルと報告形式

### 基本条件
- **比較条件**: 「Style報酬なし（ベースライン）」vs「Style報酬あり」
- **評価環境**: スタイルの視認性が高い平面環境。
- **統計処理**: 3〜5 seed の平均±標準誤差。
- **トレードオフ分析**: スタイル報酬の強さ（$\beta$）を変えた際の「タスク成功率 vs Style指標」の Pareto図 を作成。

## 6. 今後の課題：未知オノマトペ（Open-set）への対応
- **既知スタイルへの最近傍写像**で対応する。
- コサイン類似度が閾値未満の場合は "Unknown" として扱い、無理なスタイル付与を避ける仕組みを検討。

### 分析・考察
1.  **全体的な歩行速度の低下**:
    *   全スタイルにおいて `Vel X` が 0.08 m/s 未満と非常に遅い（ほぼ足踏み状態）。
    *   スタイル報酬の重みが強すぎる、またはスタイル表現と前方移動がコンフリクトを起こしている可能性がある。
2.  **成功しているスタイル**:
    *   「よろよろ」「ぶらぶら」といった**不安定・揺れる**動作は、高い角度速度（AngVel Z）と類似度スコアが出ており、モデルが特徴を捉えている。
3.  **失敗しているスタイル**:
    *   「せかせか」は類似度が低く、速い動きやピッチの細かい動きが再現できていない。
