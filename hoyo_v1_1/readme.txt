HOYO　(ver. 1.1) [2021/12/3更新]

内容物

○○.json

+ id     : シーケンスのIDです．
+ person : 歩行者の識別子です（0～9の10人）．
+ view   : 歩行者の向きです（frontまたはback）．
+ annotation
|  + instruction : 撮影時に教示したオノマトペです．
|  + select      : 10種選択式の被験者実験[1]の結果を表すベクトルです．
|  |               それぞれ[すたすた，せかせか，てくてく，どっしどっし，とぼとぼ，のしのし，のろのろ，ぶらぶら，よたよた，よろよろ]の得票数を表します．
|  + freestyle   : 自由記述式の被験者実験[2]の結果です．無効回答は除外されています．
+phase   :歩容の位相情報です．
|  + initial : 最初の1歩が右足からか左足からかを表します（rightまたはleft）．
|  + turning : 足が設置する瞬間のフレーム番号を列挙したリストです．
+length  : シーケンスの映像長です．
+path    : 部位座標データの相対ファイルパスです．


data/○○.pickle

python(3.x系)でpickleしています．
中身は(映像長,14,2)サイズのnumpy.arrayです．
部位座標の並びは[頭，首，右肩，右肘，右手，左肩，左肘，左手，右腰，右膝，右足，左腰，左膝，左足]の順で，座標値は[y,x]の順で格納されています．



[1] 加藤ら，"音象徴性を利用したオノマトペによる歩容の記述"，人工知能学会論文誌, 33巻4号, p.B-HC2_1-9, 2018.
[2] 加藤ら, "HOYO：オノマトペを付与した歩容データセット"，電子情報通信学会メディアエクスペリエンス・バーチャル環境基礎研究会（MVE）, MVE2018-21, 2018.
