# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**NaVILA-Bench** - オノマトペによるロボット歩行スタイル制御ベンチマーク

### プロダクトコンセプト
- **直感的な言語指示**: 日本語オノマトペ（「すたすた」「よろよろ」など）を使った自然な歩行指示
- **対照学習ベースの埋め込み**: MotionCLIPアーキテクチャによるテキスト-モーション空間の整列
- **強化学習との統合**: PPOベースのハイブリッド残差ポリシーによるスタイル付き歩行獲得

### プロダクトビジョン
ロボティクス研究者が、オノマトペという直感的な言語インターフェースを通じて、ヒューマノイドロボットに多様な歩行スタイルを学習させることができるベンチマークフレームワークを提供する。

従来の数値パラメータによる制御ではなく、人間が感覚的に理解できるオノマトペを用いることで、ロボットの動作設計をより自然で直感的なものにし、ロボットと人間のコミュニケーションの新たな可能性を開く。

**現状**: 対照学習と強化学習の基本実装は完了。現在は実験・評価・最適化フェーズ。

### 目的
- オノマトペとモーションの対照学習モデルの性能を定量評価し、改善する
- 強化学習によるスタイル付き歩行ポリシーのパラメータを最適化する
- 再現性のある実験プロトコルと評価基準を確立する
- 日本語オノマトペの音象徴性をロボット制御に活用する研究の知見を蓄積する

## 成功指標(KPI)

### プライマリーKPI
- **対照学習モデル精度（11クラス分類）**: 40%以上（3ヶ月後）
  - 測定方法: テストセット（65サンプル）での分類精度
  - 現状: Sarashina 44.6%を達成
- **強化学習の安定性**: Mean Episode Length 900 steps以上（転倒せずに歩行継続）
  - 測定方法: 2000イテレーション学習後のエピソード平均長
  - 現状: 970.9 stepsを達成
- **スタイル再現度（テキスト類似度）**: 0.05以上
  - 測定方法: cos(z_motion, z_text)の平均値
  - 現状: 0.025（改善の余地あり）

### セカンダリーKPI
- **リポジトリスター数**: 100（6ヶ月後）
- **論文引用数**: 10（1年後）
- **コミュニティからのIssue/PR**: 月5件以上（3ヶ月後）
- **チュートリアル完了率**: ドキュメントを見て環境構築できた人の割合80%以上

## 機能要件

**注**: 基本実装は完了。以下は実験・評価・最適化フェーズでの要件。

### コア機能（現在進行中）

#### 1. 対照学習モデルのパラメータ最適化と再実験

**ユーザーストーリー**:
ロボティクス研究者として、オノマトペとモーションの対応精度を向上させるために、複数のハイパーパラメータ設定で再実験したい

**受け入れ条件**:
- [x] HoYoデータセット（292サンプル、11種類オノマトペ）を読み込める
- [x] MotionCLIPアーキテクチャ（VAE + Supervised Contrastive Learning）で学習できる
- [x] Sarashina、SigLIPなど複数のテキストエンコーダを選択できる
- [x] 学習曲線（Loss、Accuracy、Silhouette Score）がW&Bにロギングされる
- [x] 学習済みモデル（checkpoint）が保存される
- [ ] **パラメータグリッドサーチ**: lambda_contrastive（0.1, 0.5, 1.0）、temp（0.05, 0.07, 0.1）で実験
- [ ] **学習ステップ数の影響調査**: 3000, 5000, 10000ステップでの精度比較
- [ ] **データ拡張の効果検証**: ノイズ追加、時間軸の伸縮などの効果を測定

**現状**: Sarashina 44.6%（11クラス）、SigLIP 24.6%を達成
**目標**: 50%以上（11クラス）、75%以上（4クラス）

**優先度**: P0（必須）

#### 2. 強化学習のスタイル報酬重みの最適化

**ユーザーストーリー**:
ロボティクス研究者として、スタイル表現と歩行安定性のバランスを最適化するために、報酬重みを系統的に調整したい

**受け入れ条件**:
- [x] Isaac Sim環境でH1ロボットをシミュレーションできる
- [x] PPOアルゴリズム（RSL-RL）で学習できる
- [x] ハイブリッド残差ポリシー（Base + Residual）を使用できる
- [x] スタイル報酬（cos(z_motion, z_text)）が計算される
- [x] 基本歩行報酬（速度追従、姿勢安定性など）と組み合わせて学習できる
- [x] 学習済みポリシーで推論（play.py）ができる
- [ ] **スタイル報酬重みの実験**: style_weight（1.0, 3.0, 5.0, 10.0）での比較
- [ ] **Residual Scaleの実験**: 0.1, 0.3, 0.5での表現力と安定性のトレードオフ評価
- [ ] **ベース報酬の緩和実験**: feet_air_time、flat_orientation の重みを調整
- [ ] **学習ステップ数の延長**: 2000 → 5000イテレーションでの収束性確認

**現状**: TextSim 0.025（低い）、Episode Length 970.9（良好）
**目標**: TextSim 0.10以上、Episode Length 900以上を維持

**優先度**: P0（必須）

#### 3. 定量評価プロトコルの拡充と実行

**ユーザーストーリー**:
研究者として、モデルの性能を多角的に評価し、論文に掲載できる定量データを取得したい

**受け入れ条件**:
- [x] 対照学習の評価:
  - [x] Retrieval指標（R@1, R@3, R@5, MedR）を計算できる
  - [x] Clustering指標（Silhouette Score）を計算できる
  - [x] Fine（11クラス）とCoarse（4クラス）の両方で評価できる
- [x] 強化学習の評価:
  - [x] スタイル類似度（cos(z_motion, z_teacher)）を計算できる
  - [x] 転倒率を測定できる
  - [x] 速度追従誤差を測定できる
- [x] 混同行列、PCA可視化を生成できる
- [ ] **HOYOデータセットとの誤差評価**: DTWによる位相合わせ後の関節位置・速度誤差
- [ ] **人間評価インターフェース**: 生成モーションを見て、どのオノマトペに近いかを人間が評価（N≥10人）
- [ ] **オノマトペ間の距離行列**: 11×11の類似度行列を計算し、階層的クラスタリング
- [ ] **評価結果のCSV出力**: 全実験の定量結果を比較可能な形式で保存

**優先度**: P0（必須）

## CLIインターフェース

### 対照学習の学習
```bash
# Sarashina エンコーダで学習（推奨）
/home/jouta/venvs/motionclip/bin/python hoyo_v1_1/models/train_motionclip_joint.py \
  --stage full \
  --sem-encoder sarashina \
  --label-mode fine \
  --steps 5000 \
  --batch-size 32 \
  --lr 5e-5 \
  --lr-encoder 2e-5 \
  --lr-decoder 2e-5 \
  --lambda-vae 1.0 \
  --lambda-contrastive 0.5 \
  --temp 0.07 \
  --seed 42 \
  --run-name my_experiment
```

## 非機能要件

### パフォーマンス
- 対照学習の学習時間: 5000ステップを24GB VRAM GPUで12時間以内
- 強化学習の学習時間: 2000イテレーション（2048環境並列）を24GB VRAM GPUで48時間以内
- 推論速度: リアルタイム（50Hz）でポリシー実行が可能

### ユーザビリティ
- 新規ユーザーがREADMEに従って環境構築を完了できる（60分以内）
- 実験設定ファイル（YAML/Python Config）が明確に構造化されている
- W&Bなどのロギングツールで学習曲線をリアルタイムで確認できる

### 信頼性
- 学習途中でクラッシュした場合、チェックポイントから再開できる
- データセットの破損を検出し、警告を表示する
- 再現性: 同じシード値で同じ結果を再現できる（±5%の誤差範囲）

### 拡張性
- 新しいオノマトペを追加できる（データセットの拡張）
- 新しいテキストエンコーダを追加できる（モジュール化されたエンコーダインターフェース）
- 新しい報酬関数を追加できる（プラグイン形式の報酬設計）

### 互換性
- Python 3.10以上
- CUDA 11.8以上
- Isaac Sim 4.5.0
- Isaac Lab 2.0.2
- Ubuntu 20.04以上

## スコープ外

明示的にスコープ外とする項目:
- 実機ロボットでの動作（シミュレーションのみ）
- リアルタイム音声認識からのオノマトペ抽出
- 歩行以外の動作（走行、ジャンプ、上肢動作など）
- 商用利用向けの最適化
- GUIベースの操作インターフェース（CLIのみ）
